工厂方法模式:Factory Method Pattern

1 工厂方法模式可以解决简单工厂模式当中的0CP问题。
    怎么解决的?
    一个工厂对应生产一种产品。
    这样工厂就不是全能类了，不是上帝类了。
    另外，也可以符合0CP原则。
2工厂方法模式中的角色:
    *抽象产品角色 Weapon
    *具体产品角色 Fighter Gun
    *抽象工厂角色 WeaponFactory
    *具体工厂角色 FighterFactory GunFactory
工厂方法模式的优点:
当你扩展一个产品的时候，符合0CP原则，因为只害要添加两个类，
一个类是具体产品类，一个类是且体工厂类。都是添加类，没有修改之前的代码，所以符合00E
工厂方法模式的缺点:4.
每次增加一个产品时，都需要增加一个具体类和对象实现工厂，使得系统中类的个数成倍增加，
在一定程度上增加了系统的复杂度，同时也增加了系统具体类的依赖。这并不是什么好事。

=======================================================================================================
总结：简单工厂模式当我们需要添加一个产品的时候，我们需要在工厂的判断逻辑内部添加判断代码 不符合开闭原则
    而工厂方法模式，我们只需要添加类和类对应工厂继承对应抽象类，然后在主代码中不改变原代码，
    在原代码的下面添加代码，符合开闭原则

    --------------
    开闭原则：添加新功能的时候，添加功能的时候，开放代码扩展，关闭对原代码和代码逻辑的修改
            添加新功能的时候，不改变原代码的逻辑，直接在原代码下面写新代码
